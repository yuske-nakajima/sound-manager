# 調査スキル

## 目的
コードベースの調査、技術調査、問題の根本原因分析などを行い、結果を体系的にドキュメント化する。

## 適用ケース

### 必ず使用するケース
- バグの原因調査
- 既存機能の動作確認
- コードベースの構造把握
- 技術的な実現可能性の調査
- パフォーマンス問題の分析
- セキュリティリスクの調査

### 使用しないケース
- 単純なファイル読み込み
- 1-2ファイルの確認のみ

## 調査プロセス

### 1. 調査範囲の明確化
- 何を調べるのかを明確にする
- 調査の深さを決める（quick/medium/thorough）
- 期待する成果物を定義

### 2. 調査の実施

#### Explore subagent の活用
複数ファイルにまたがる調査や、コードベース全体の把握が必要な場合:
```
Task tool with subagent_type='Explore'
```

#### Serena MCP の活用
特定のシンボルやパターンの調査:
- `mcp__serena__get_symbols_overview`: ファイル構造の把握
- `mcp__serena__find_symbol`: シンボル検索
- `mcp__serena__search_for_pattern`: パターン検索
- `mcp__serena__find_referencing_symbols`: 参照元の検索

#### その他のツール
- `Read`: ファイル内容の確認
- `Grep`: コード内のパターン検索
- `Glob`: ファイル検索

### 3. 成果物の作成

**調査結果の作成**
- 調査結果を詳細に記述して返す
- 以下の内容を含める:
  - 調査日時
  - 調査の目的・背景
  - 調査対象（ファイルパス、関数名など）
  - 調査結果（発見事項）
  - 問題点・課題
  - 次のステップ（推奨アクション）
- ファイル保存は**メインスレッドが実施**するため、subagent は保存不要

## 調査結果のテンプレート

```markdown
# [調査タイトル]

## 調査日時
YYYY-MM-DD

## 調査の目的・背景
[なぜこの調査が必要だったか]

## 調査対象
- ファイル: [ファイルパス]
- 関数/クラス: [対象のシンボル]

## 調査方法
- 使用したツール: [Explore subagent / Serena MCP / Read など]
- 調査範囲: [quick/medium/thorough]

## 調査結果

### 発見事項
[調査で分かったこと]

### 問題点・課題
[見つかった問題]

### 根本原因（該当する場合）
[問題の根本原因]

## 影響範囲
[この問題の影響を受ける箇所]

## 次のステップ
[推奨されるアクション]
- [ ] アクション1
- [ ] アクション2

## 参考資料
- 関連ファイル: [ファイルパス]
- 関連ドキュメント: [ドキュメントへのリンク]
```

## 注意事項

### 調査の深さ
- **quick**: 基本的な確認（1-2ファイル程度）
- **medium**: 標準的な調査（複数ファイル、関連する処理の確認）
- **thorough**: 徹底的な調査（コードベース全体、依存関係の完全な把握）

### Explore subagent 使用時の注意
- subagent が返した結果は、そのままではなく整理して保存する
- 重要な発見は強調して記載
- ファイルパスや行番号は正確に記載

### 調査結果の品質
- 推測と事実を明確に区別する
- コードの引用は必要最小限にする
- 図やテーブルを活用して分かりやすくする

## 調査完了のチェックリスト

調査を完了する前に以下を確認:
- [ ] 調査の目的を達成したか
- [ ] 発見事項を明確に記載したか
- [ ] 問題点を特定したか（該当する場合）
- [ ] 次のステップを提示したか
- [ ] 調査結果を詳細に返したか（保存はメインスレッドが実施）

## 関連スキル
- `planning`: 調査結果を元に実装計画を立案
- `implementation`: 調査結果を元に実装
- `review`: 調査結果を元にレビュー
